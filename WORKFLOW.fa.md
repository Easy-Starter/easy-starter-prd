# فرایند: از ایده تا لانچ

## ۰. دریافت ایده

خروجی‌ها: **خلاصه یک‌صفحه‌ای، لاگ فرضیات، سوالات باز**

معیار خروج: **مسئله، کاربر هدف، خروجی مطلوب و ابهام‌ها مشخص‌اند.**

## ۱. دیسکاوری

خروجی‌ها: **پلن تحقیق، نوت مصاحبه، تحلیل جایگزین‌ها، پرسونا/JTBD، سفر کاربر**

معیار خروج: **درد کاربر با شواهد پشتیبانی شده یا به‌عنوان فرضیه مشخص است.**

## ۲. تعریف محصول

خروجی‌ها: **PRD، محدوده، متریک‌های موفقیت، اولویت‌بندی**

معیار خروج: **MVP و معیارهای پذیرش قابل تست هستند.**

## ۳. طراحی

خروجی‌ها: **Design Brief، معماری اطلاعات، فلوها، UX/UI Spec، پروتوتایپ، پلن تست کاربردپذیری**

معیار خروج: **happy path، edge case، حالت‌های empty/loading/error، responsive و accessibility پوشش داده شده‌اند.**

## ۴. برنامه‌ریزی فنی

خروجی‌ها: **Technical Brief، API/Data Spec، Security/Privacy Brief، Feature Spec، Plan**

معیار خروج: **معماری، قراردادها، مدل داده، تست و rollback مشخص است.**

## ۵. تسک‌سازی

خروجی‌ها: **tasks.md و QA plan**

معیار خروج: **تسک‌ها کوچک، مرتب، تست‌محور و دارای requirement ID هستند.**

## ۶. پیاده‌سازی

خروجی‌ها: **branch، PR، تست، changelog**

معیار خروج: **تست‌ها پاس می‌شوند و PR به requirement ID وصل است.**

## ۷. لانچ

خروجی‌ها: **پلن ریلیز، GTM، آنالیتیکس، مستندات پشتیبانی، rollback**

معیار خروج: **ریلیز قابل اندازه‌گیری و قابل برگشت است.**

## ۸. یادگیری

خروجی‌ها: **گزارش متریک، فیدبک، تصمیمات، specهای به‌روز**

معیار خروج: **یادگیری به تصمیم محصولی تبدیل شده است**
