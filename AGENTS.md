# Agent Instructions

1. Read the relevant PRD, Design Brief, Feature Spec, Plan, and Tasks before coding.
2. Do not add features not requested by the spec.
3. If requirements conflict, report the conflict before implementation.
4. Preserve requirement IDs in summaries, commits, and pull requests.
5. Prefer simple maintainable solutions over premature abstractions.
6. Write/update tests before implementation where possible.
7. Update documentation when product behavior changes.
8. Never store secrets in source control.
9. Final summaries must include completed tasks, requirement IDs covered, tests run, docs updated, and follow-ups.
